Commands: [required] <optional>
/realtor
Displays a list of player commands
/realtor buy [region] <player>
Buy a buyable region. The money you pay goes to the current owner, you become the new owner, and the region becomes no longer buyable.
With permission stealthrealtor.force: If the player argument is supplied, it forces the transaction for another player instead of you.
/realtor rent [region] [days] <player>
Rent a rentable region for a number of real-life days, if it isn't already being rented. The entire rental cost is paid up front. The money you pay goes to the owner (or if no owner, to the owner of the parent region), and you become a member for the duration of your rent. Use this on a region you currently rent to extend it by the given number of days.
With permission stealthrealtor.force: If the player argument is supplied, it forces the transaction for another player instead of you.
/realtor info [region]
List everything StealthRealtor knows about this region including time left on current rental and its interpretation of the flags.
Also, if there is any inconsistency between the region and database regarding rentals, the info command will fix it and log what it did.
/realtor list available <buy|rent>
List all regions available to buy or rent.
/realtor list within [region] <buy|rent>
List all regions available to buy or rent that physically overlap or are contained by the named region. Useful when there are a lot and you want to look locally.
/realtor list rented <player>
List regions you are currently renting.
With permission stealthrealtor.spy: If the player argument is supplied, you can see another player's current rentals.
/realtor reload
Reload the plugin.
Requires permission stealthrealtor.reload
/realtor version
Show curent version.

How to set up regions for buying and renting:
In WorldGuard, set the 'buyable' flag for regions to be sold outright. Non-buyable regions will be considered rentable instead. Then set the 'price' flag -- for buyable regions this is just the price, but for rentable regions this is the per-day cost of renting.

Set the region owner to determine who will recieve the money. For rentable regions, you can leave it blank and set the owner of the parent region instead, that way you can have an "Inn" region owned by the proprietor, parent to all the rooms, and not have to set the owner each time.

But either way you MUST have an owner defined somewhere so the plugin knows who to give the money to. Otherwise it will not allow anyone to buy or rent it. If you really want the money to go to nobody, you can enter a fictitious name.
